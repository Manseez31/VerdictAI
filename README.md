## вҡ пёҸ Disclaimer вҖ” Not Legal Advice

VerdictAI is an **educational and research tool**. Every output вҖ” answers,
citations, case analyses, simulated prosecution/defense arguments, and simulated
verdicts вҖ” is generated by AI language models and may be incomplete, outdated, or
wrong. It is **not legal advice**, creates **no lawyer-client relationship**, and
is **not a prediction of any real court outcome**. Always verify against the
primary legal texts and consult a qualified advocate for any real legal matter.

VerdictAI will **not** assist with committing crimes, evading law enforcement,
destroying evidence, or any other unlawful conduct.

## Features

### рҹ”Қ Legal Q&A (RAG)
- **Hybrid retrieval** вҖ” dense embeddings (multilingual MPNet) + **BM25** lexical
  search, fused with **Reciprocal Rank Fusion**. Measurably better than dense-only:
  **+13% Recall, +16% Context Precision**, at no latency cost.
- **Query rewriting** вҖ” rewrites questions into retrieval-optimized queries,
  expanding legal terminology (e.g. *khop* вҶ’ аӨӘаҘҚаӨ°аӨӨаӨҝаӨ°аӨ•аҘҚаӨ·аӨЈ) and transliterating
  romanized Nepali to Devanagari.
- **Arena routing** вҖ” routes each question to the right Act via keyword rules
  with a TF-IDF + Logistic Regression classifier fallback.
- **Citations everywhere** вҖ” every answer is grounded in and cited to the source
  Act and section (`[аӨёаҘҚаӨ°аҘӢаӨӨ: вҖҰ]`), rendered as clickable citation cards.
- **Independent LLM judge** вҖ” a separate, larger model scores each answer's
  correctness and faithfulness against the retrieved context (no self-grading).
- **Bilingual** вҖ” answers generate in Nepali; one click gives a professional
  English translation with citations preserved. Cached, so toggling is instant.

### рҹ§  Legal Case Intelligence Suite
Eight specialist agents turn a case scenario (typed, or uploaded as **PDF/DOCX**)
into a full analysis dashboard:

| Agent | Produces |
|-------|----------|
| **Case Analyzer** | parties, dates, locations, alleged actions, evidence, legal issues, category |
| **Evidence Analyzer** | per-item Strong/Moderate/Weak + **missing** evidence, with reasoning |
| **Legal Research** | рҹ”— **RAG-grounded** вҖ” retrieves applicable Acts/sections + real citations |
| **Prosecutor** | strongest lawful case, evidence-based arguments, aggravating factors |
| **Defense** | strongest lawful defense, prosecution weaknesses, mitigation |
| **Judge** | neutral reasoning, findings, evidence + citation-quality assessment |
| **Verdict** | likely outcome, rationale, **uncertainty analysis**, confidence |
| **Timeline** | chronological case timeline |

Plus **6 demo scenarios**, per-agent **explainability** (reasoning + laws +
confidence meter), and a **safety preamble** on every agent prompt.

### вҡ–пёҸ Case Simulator
A lightweight 5-agent moot-court simulation with **PDF report export**.

### рҹҺЁ Interface
Responsive, accessible (ARIA, keyboard nav, reduced-motion), **dark mode**,
streaming answers, and full loading/empty/error states. No build step вҖ”
vanilla ES modules + Tailwind via CDN.

## Quick Start

**Requirements:** Python 3.13+, [`uv`](https://docs.astral.sh/uv/), and a free
[Groq API key](https://console.groq.com/keys).

```bash
# 1. Clone
git clone https://github.com/Manseez31/VerdictAI.git
cd VerdictAI

# 2. Install dependencies
uv sync

# 3. Configure
cp .env.example .env
#    вҖҰthen edit .env and set GROQ_API_KEY=your_key_here

# 4. Run
uv run uvicorn backend:app --host 127.0.0.1 --port 8000
```

Open **http://127.0.0.1:8000** рҹҺү

> The vector store is committed, so there is **no indexing step** вҖ” it works
> immediately. On first run the embedding model (~1 GB) downloads automatically.

## Configuration

All settings are optional except `GROQ_API_KEY`. Set them in `.env` (see
[`.env.example`](.env.example)).

| Variable | Default | Purpose |
|----------|---------|---------|
| `GROQ_API_KEY` | *(required)* | Groq API key |
| `ALLOWED_ORIGINS` | `localhost` | CORS allow-list |
| `RATE_LIMIT_PER_MINUTE` | `20` | Per-IP request cap |
| `LOG_LEVEL` | `INFO` | Logging level |
| `ENABLE_HYBRID_RETRIEVAL` | `true` | Dense + BM25 fusion (`false` вҶ’ dense only) |
| `ENABLE_QUERY_REWRITING` | `true` | Retrieval-optimized query rewriting |
| `DENSE_WEIGHT` / `BM25_WEIGHT` | `1.0` | RRF fusion weights |
| `RRF_K` | `60` | RRF damping constant |
| `ENABLE_RERANKER` | `false` | Cross-encoder reranking (see [caveat](docs/RETRIEVAL.md)) |
| `RETRIEVAL_DEBUG` | `false` | Log per-chunk retrieval diagnostics |
| `TRANSLATION_MODEL` | `llama-3.3-70b-versatile` | Model for translation |
| `CASE_INTELLIGENCE_MODEL` | `llama-3.3-70b-versatile` | Model for the agents |

## Retrieval Quality & Benchmarks

VerdictAI ships an **offline evaluation harness** and a **benchmark** so retrieval
changes are judged on numbers, not vibes.

```bash
uv run python evaluation.py --build-gold   # (re)generate the gold set
uv run python benchmark.py --k 5           # Dense vs Hybrid vs Hybrid+Reranker
```

Results on the seed gold set (30 queries, k=5, whole corpus):

| Metric | Dense only | **Hybrid** | Hybrid + Reranker |
|--------|-----------:|-----------:|------------------:|
| Recall@5 | 0.767 | **0.867** | 0.800 |
| MRR | 0.708 | **0.786** | 0.734 |
| Context Precision@5 | 0.573 | **0.667** | 0.567 |
| Retrieval latency | 22 ms | **22 ms** | 3599 ms |

**Hybrid retrieval wins on every metric at zero latency cost.** The cross-encoder
reranker (`bge-reranker-base`) *regresses* quality on Devanagari text and is
therefore **off by default** вҖ” an honest negative result, documented in full in
[`docs/RETRIEVAL.md`](docs/RETRIEVAL.md).

## Testing

```bash
uv run pytest              # 77 tests
```

All tests run **fully offline** вҖ” LLMs and the RAG retriever are faked, so no
API key, network, or model download is needed. Coverage: retrieval fusion (RRF),
BM25 tokenization, arena routing, translation, the 8-agent pipeline, document
extraction, and every API endpoint contract.

## Roadmap
